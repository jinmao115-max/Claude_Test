# 27卒就職比較表 作成手順書

就職先比較表（Excel / PDF）を、**同じ手順で繰り返し更新できる**ための手順書です。  
データの正本（マスター）を `create_6company_comparison.py` に置き、そこから2形式を一括生成します。

> **2026年6月更新（最新）**: 山陽色素・極東開発工業を除外し6社構成に変更。  
> 並び順: 伯東・ダイニック・ヤマトプロテック・ラサ工業・阪神内燃機工業・SWCC。  
> 「工場・研究・開発拠点」行に関西エリア拠点の事業内容サマリーを追記。


## 4. データ調査（出典の優先順位）

数値・制度は、次の順で確認し、セルに **出典と時点** を書く。  
**情報が競合する場合は左の優先度が高い方を採用すること。**

| 優先 | 出典 | 主な取得項目 |
|------|------|-------------|
| 1 | 各社 **公式採用サイト** | 初任給（月額）・賞与・年間休日・社宅・募集職種 |
| 2 | [マイナビ2027](https://job.mynavi.jp/27/) 各社ページ | 資本金・売上・従業員・残業・有給・採用実績校 |
| 3 | [OpenWork](https://www.openwork.jp/) | 総合評価・待遇・士気・20代成長・平均年収・残業・有給 |
| 4 | [kabutan/irbank](https://irbank.net/) 業績データ | 売上高・営業利益・業績トレンド（上場企業） |
| 5 | [indeed](https://jp.indeed.com/) | 職種別年収・口コミ・働きやすさ |
| 6 | 日経会社情報・決算短信・IR | 売上・利益の補足、株価 |

### 4.1 マイナビ URL の調べ方

1. マイナビ2027で企業名を検索
2. 会社概要ページの URL から `corpXXXX` を確認  
   例: `https://job.mynavi.jp/27/pc/search/corp55663/outline.html` → corp55663
3. スクリプトの `URLS` リストに登録する

### 4.2 OpenWork スコアの取得方法

OpenWork の企業ページを参照する。  
取得すべきスコアは**必ず以下を全て確認**すること（一部だけ確認すると漏れが発生する）。

| 取得項目 | スクリプト内の行ラベル |
|----------|----------------------|
| 総合評価（5点満点・回答者数） | `OpenWork総合評価(5点)` |
| 待遇面の満足度 | `待遇面の満足度` |
| 社員の士気 | `社員の士気` |
| 20代成長環境 | `20代成長環境` |
| 平均年収（OW）・回答者数 | `平均年収(会社全体)` |
| 月間残業時間 | `残業時間(月・OpenWork)` |
| 有給消化率 | `有給休暇消化率` |

> **過去の失敗事例（2026年6月）**: 調査プロンプトが「一部スコアのみ」を求める不完全な指示だったため、  
> ラサ工業の「待遇面の満足度」が欠落した（正しくは 3.1）。  
> **対策**: 上記テーブルの全項目を明示的に指定してフェッチすること。

### 4.3 kabutan/irbank での業績確認

上場企業の最新業績は irbank または kabutan から取得する。

```
https://irbank.net/E02802/pl  ← 伯東(7433)
https://irbank.net/E01336/pl  ← SWCC(5805)
https://kabutan.jp/stock/finance?code=XXXX
```

kabutan は直接 WebFetch ができない（403）ため、**WebSearch で「XXXX 売上高 営業利益 最新決算」**として検索する。

### 4.4 記載ルール

- **単位・期を明記**: 例 `連結477億円(2026/3期)`、`84億4,300万円`
- **連結 / 単体** を混同しない
- マイナビと OpenWork で残業時間が違う場合は **両方の行** に分けて記載
- 非公表は `非公表`、不明は `—`
- 公式 HP と第三者サイトで数値が違う場合は **公式を優先**し、注釈を付ける

## 5. スクリプトの編集

編集対象: `create_7company_comparison.py`

### 5.1 データ構造

スクリプトには3種類の行定義がある。

| 種類 | 書き方 | 用途 |
|------|--------|------|
| セクション見出し | `("【基本情報】", None)` | 青帯の見出し行 |
| 通常データ行 | `("初任給(大卒・月)", ["値1", "値2", ...])` | 各企業の個別値 |
| ざっくり比較行 | `("初任給（大卒・月給）", "全社まとめ文章")` | 値が全列をまたぐ横結合行 |
| 空行 | `("", None)` | セクション間の区切り |

### 5.2 企業の追加・削除・並び替え

**必ず同時に直す箇所**（列数がずれると生成失敗）:

| 定数 | 内容 |
|------|------|
| `CN` | 企業名リスト（列順＝表示順） |
| `ROWS` | 各タプルの `values` リスト（企業数と同じ長さ） |
| `AXIS_ROWS` | ざっくり比較の文言（企業名の入れ替え・削除） |
| `URLS` | 参照URLシート用 |

**チェック**: 全 `ROWS` の通常行で `len(values) == len(CN)` であること。

### 5.3 社数が変わったときのファイル名

スクリプト冒頭の定数を社数に合わせて変更する。

```python
SHEET_MAIN = "7社比較"         # ← 社数に合わせる
OUT_XLSX  = os.path.join(OUT_DIR, "27卒_7社就職比較表.xlsx")
OUT_LOCAL = r"C:\...\Temp\27卒_7社就職比較表.pdf"
OUT_PDF   = os.path.join(OUT_DIR, "27卒_7社就職比較表.pdf")
```

実行すると以下が出力される。


- `Table natural size` が `avail` より大きい場合、自動スケーリングで1ページに収める
- `Scale` が 0.7 を下回る場合はフォントが小さすぎるため、フォントサイズ `fs_val` を下げるか行数を減らすことを検討

### 6.1 目視確認のポイント

| 確認項目 | 方法 |
|----------|------|
| 列数・社名 | Excel の1行目 |
| 追加した行 | Excel で該当項目を検索 |
| PDF 1枚収まり | PDF を開いてページ数確認 |
| 文字の重なりなし | PDF の各セルでテキストが収まっているか目視 |
| 削除企業が残っていない | PDF 内検索（山陽色素が入っていないか） |
| 数値の文字化け | PDF で数字・円・% が欠けていないか |
| ざっくり比較の横結合 | PDF で比較行がセル全幅に表示されているか |

## 7. 比較項目一覧（現行テンプレート）

セクションごとの行ラベル一覧。行を増やすときはこの一覧をベースにする。

**【基本情報】**  
正式社名 / 上場区分 / 資本金 / 本社所在地 / 従業員数 / 創業・歴史

**【財務・業績】**  
最新売上高 / 最新営業利益 / 過去10年の業績トレンド / 株価推移(10年)

**【勤務地・拠点(営業所除く)】**  
理系・技術の主な勤務地 / 工場・研究拠点(例) / 27卒主な募集職種

**【OpenWork・働き方】**  
OpenWork総合評価 / 待遇面の満足度 / 社員の士気 / 20代成長環境 / 平均年収 / 30歳前後の年収目安 / 有給休暇消化率 / 残業時間(マイナビ/公式) / 残業時間(OpenWork) / 平均勤続年数

**【給与・福利厚生】**  
初任給 / 賞与実績 / 年間休日 / 住宅手当・社宅 / 食事補助 / 資格取得支援 / その他特色制度 / 育児休業取得率(男性)

**【27卒・就活メモ】**  
事業の軸 / マイナビ2027 / 採用実績に大阪電気通信大学 / データ出典・更新

**【ざっくり比較（就活の軸）】**（`AXIS_ROWS` / 横結合）  
初任給 / 資本金 / 会社規模・売上 / 口コミ総合 / 残業 / 賞与 / 専門性 / 社会貢献 / 上場・将来性 / 出典

## 8. トラブルシューティング

| 症状 | 原因 | 対処 |
|------|------|------|
| `Permission denied` でPDFコピー失敗 | ネットワーク先のPDFが開かれている | PDFを閉じてから再実行 or ローカルの `OUT_LOCAL` を手動コピー |
| PDF で文字が重なる | 行高さが固定値になっている | `rowHeights` を指定せず自動サイズにする（現行スクリプトは対応済み） |
| PDF が2ページ以上になる | Scale が小さすぎる・行数が多すぎる | フォントサイズ `fs_val` を下げるか行を削減 |
| `列ずれエラー` / 生成が崩れる | `values` の要素数 ≠ `len(CN)` | 全 `ROWS` 行の要素数を確認 |
| OpenWork スコアに欠落 | 調査時に全スコアを明示指定しなかった | §4.2 の全項目リストを使って再取得 |
| kabutan が 403 で取得できない | WebFetch ブロック | WebSearch で「会社名 証券コード 売上高 最新決算」で代替 |
